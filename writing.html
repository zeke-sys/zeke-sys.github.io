Writing & Notes

Short, practical pieces on how I think about systems, security, and code quality.

What audit taught me about code reviews

Years in IT audit shaped how I approach reviews: evidence over assumptions, clear traceability, and a focus on real risk.

In code, that translates to small pull requests, explicit reasoning in descriptions, and tests that prove the behavior we care about.

Building a trie autocomplete from scratch

Implementing a trie pushed me to think carefully about memory layout, traversal patterns, and how user behavior influences data structures.

The browser demo on this site mirrors the C++ implementation and makes those ideas visible as you type.

Bringing security thinking into early designs

Security is most effective when it's part of the first draft, not a bolt-on. I like to ask, "What can go wrong here?" before writing code.

That mindset helps catch issues in logging, error handling, and data flows long before they reach production.
